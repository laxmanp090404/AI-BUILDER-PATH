# Local Retrieval-Augmented Generation (RAG) System

A fully offline, local Retrieval-Augmented Generation (RAG) system built with **Python**, **LangChain**, **ChromaDB**, and **Ollama**. This system allows you to index local documents and query them privately without sending any data to external cloud services.

## Workflow & Architecture

### VIEW ARCHITECTURE DIAGRAM

[Architecture Diagram](./archdiagram.png)

The system operates entirely on your local machine using the following pipeline:

1. **Document Discovery & Loading**: Automatically scans `data/raw/` for supported files (`.txt`, `.md`, `.pdf`, `.docx`), loads them, and converts them to LangChain Document objects.
2. **Chunking & Metadata Addition**: Splits long text into smaller chunks using overlapping windowing. Each chunk is tagged with detailed metadata:
   - `source`: File filename (e.g. `sample.txt`).
   - `file_type`: Lowercased file extension (e.g. `.txt`).
   - `page`: 1-based page number (where applicable, e.g. for PDFs).
   - `chunk_id`: 1-based incremental chunk index specific to the source document.
3. **Data Processing Cache**: Serializes the generated chunks grouped by source document and stores them in `data/processed/` as JSON files.
4. **Vector Storage**: Generates embeddings using `nomic-embed-text` and indexes them in ChromaDB (persisted locally inside the `vector_store/` directory).
5. **Retrieval**: Performs semantic similarity searches based on user queries, retrieving the top-K relevant chunks.
6. **Restricted Generation**: Incorporates the retrieved chunks into a prompt. The LLM (e.g., `qwen3.5:4b` or `deepseek-r1:7b`) uses only the provided context. If the query is a simple greeting (e.g. "hi"), it responds politely. If the query cannot be answered using the retrieved documents, it replies exactly with: `"I could not find that information in the provided documents."`

---

## Key Features

- **Off-Line (Fully Offline)**: Zero network calls; all embeddings and generation steps happen on your local consumer hardware.
- **Smart KB Setup**: Detects if a local database already exists and allows the user to either **Reuse** it (skipping re-embedding) or **Rebuild** it (regenerates embeddings from raw data).
- **Thinking Tag Scrubbing**: Automatically detects and strips `<thought>` tag wrappers generated by reasoning models like DeepSeek-R1, leaving a clean response for the CLI interface.
- **Resilient Loaders**: Handles load errors on corrupt or unsupported file formats gracefully, printing a warning and continuing with the remaining valid files.

## Prerequisites

1. Install **Python 3.10+** (Python 3.14 recommended).
2. Install **Ollama** by following the instructions at [ollama.com](https://ollama.com).
3. Pull the required models in your local environment:
   ```bash
   ollama pull nomic-embed-text
   ollama pull qwen3.5:4b  # or deepseek-r1:7b
   ```

---

## Setup and Installation

1. Clone or navigate into the project directory:
   ```bash
   cd RAG-System
   ```
2. Activate your python virtual environment:
   ```bash
   source .venv/bin/activate
   ```
3. Install the dependencies:
   ```bash
   pip install -r requirements.txt
   ```

---

## Running the Application

1. Place your reference documents in the `data/raw/` folder (e.g. `.txt`, `.md`, `.pdf`, `.docx`).
2. Run the application:
   ```bash
   python app/main.py
   ```
3. Choose to build/rebuild the database:
   - On first run, it will automatically process the documents and build the database.
   - On subsequent runs, select `1` to **Reuse** the database (saves time) or `2` to **Rebuild** the database (incorporating new documents added to `data/raw`).
4. Type in your questions and hit **Enter**.
5. Type `exit` to exit the CLI.
